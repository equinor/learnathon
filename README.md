## 🎯 Main Event: EDC 2026

| Detail | Info |
|--------|------|
| **Date** | March 18, 2026 |
| **Duration** | Full day (7 hours) |
| **Attendees** | 60 people |
| **Format** | Guided curriculum with coached labs |
| **Venue** | One big room |

### What Participants Build
Teams create and deploy a lightweight prototype of their choice (app, automation, a new teammate... creativity encouraged!) in a preconfigured sandbox.

### The Workflow
`Mini-spec → Agent plan → Small commits → Continuous verification → Security checkpoint → Ship & share`

---

## Why?

- **Learn by doing** — not just watching presentations
- **Explore new tools** — Copilot, Claude Code, and more
- **Practice secure workflows** — security built in, not bolted on
- **Build community** — share experiences, learn from each other
- **Take it home** — repeatable patterns you can use with your team

## 🎤 Show & Tell: The Heart of the Learnathon

This is where the magic happens. Teams don't just demo — they **share what they learned**.

### Format

| Part | Time |
|------|------|
| Demo your creation | 2 min |
| Share your learnings | 2 min |
| Audience Q&A | 1-2 min |
| **Total per team** | **5-6 min** |

With ~20 teams and 1h 30m, we have room for all teams + buffer time.

### What to Share

Encourage teams to answer:
- 🧠 **What surprised you?** — Something AI did unexpectedly well (or badly)
- 🔧 **What technique worked?** — A method others should try
- ⚠️ **What would you do differently?** — Honest reflection
- 💡 **What will you take back to your team?** — Practical takeaway

### Facilitation Tips
- Have a visible timer
- Celebrate learnings as much as polished demos
- "Best incomplete project with great insights" > "Polished but nothing learned"

## 🔐 Security (Built In, Not Bolted On)

Security is woven throughout the day, not an afterthought. Teams practice secure agentic coding patterns as they build.

### Security Checkpoint (Full-Day Format)

Mid-day pause where teams review their work for security considerations before the final sprint.

**Format options** *(to be decided)*:
- Cross-team review: Teams pair up and review each other's code
- Coach review: Security-savvy facilitators do quick spot-checks
- Self-assessment: Guided checklist teams complete themselves

### Topics to Cover

> 💡 *AI-suggested areas — needs review with security experts:*
> - Secrets hygiene (no API keys in code)
> - Dependency awareness (what are you installing?)
> - Safe prompting and tool boundaries
> - Lightweight threat modeling
> - Recognizing AI hallucinations in security contexts

### Security in Awards

Recognizing good security practice is part of the awards — see "Best Safety Practice" category.

### 🔓 Sandbox Tool Access — Open Questions

> ⚠️ **Key decision for Tuesday:** Can we provide a wider set of tools in a controlled sandbox environment?

**Questions to resolve:**

1. **Isolation** — Can we set up a separate environment (Azure subscription? Isolated GitHub org?) where non-approved tools can be used safely?

2. **Data boundaries** — What restrictions do we need? (No production data, no company secrets, synthetic data only?)

3. **Tool licensing & payments**
   - ✅ **Approach:** Use existing credit card setup for tool evaluation
   - ✅ **Justification:** Capture tool evaluations during learnathon (feedback forms, learnings shared)
   - ⚠️ Access granted **only during the learnathon** — revoked after event
   - ❓ **Pricing reality:** Most tools charge monthly, not daily — need to budget for 1 month × 60 users per tool
   - ❓ Estimate total cost and get approval before event
   
   **Cost mitigation options:**
   - 🎟️ **Volume/event discounts** — ask vendors for special pricing for 60 seats
   - ⏱️ **Free trials** — time it so trials cover the event (14-day trials etc.)
   - 🤝 **Vendor sponsorship** — reach out and ask if they'd sponsor the learnathon
   - 🎯 **Limit selection** — only offer 1-2 non-approved tools to control costs

4. **Approval process** — Do we need temporary security approval for the sandbox? Who signs off?

5. **Post-event** — Tools access revoked, environment torn down

**Potential approach:**
```
┌─────────────────────────────────────────────────┐
│  Learnathon Sandbox (isolated environment)      │
│  ├── Separate Azure subscription                │
│  ├── No access to production systems            │
│  ├── Temporary tool licenses (paid via eval CC) │
│  ├── Tool evaluations captured as output        │
│  └── Torn down after event                      │
└─────────────────────────────────────────────────┘
```
