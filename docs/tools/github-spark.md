# GitHub Spark — Getting Started

GitHub Spark lets you build and deploy web apps using natural language — no local setup, no deployment config. You describe what you want, it builds and hosts it.

Access: [githubnext.com/projects/spark](https://githubnext.com/projects/spark) (requires GitHub Copilot subscription)

## What It's Good For

- Rapid web app prototypes from a description
- Sharing something with a URL immediately (no deploy step)
- Quick UIs for tools, games, or dashboards
- The "Conference Stand Game" challenge is a perfect Spark project

## How to Use It

1. Go to Spark in your browser (link above)
2. Describe what you want to build
3. Spark generates and runs the app live
4. Iterate with natural language: "add a reset button", "make the background dark"
5. Share the URL

## Tips

- Start with a clear, specific description — the more concrete, the better the first result
- Iterate in small steps rather than asking for everything at once
- If it goes off track, use the revert/checkpoint feature
- Great for the live demo — audience sees something running immediately

## Limitations

- Apps run in Spark's managed environment — you don't own the infrastructure
- Not suitable for projects that need a persistent backend or database beyond Spark's built-in capabilities
- Code is exportable but the "live" URL is tied to Spark
